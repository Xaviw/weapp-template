# 许愿池扭蛋机前端技术方案

## 目标范围

本方案覆盖“企鹅读伴”许愿池中的扭蛋机小模块，重点说明扭蛋机独立组件、许愿状态、接口与动画协同、奖品弹窗、分享入口、异常复位以及背景 Lottie 动效的实现方式。

## 实现步骤

### 1. 拆分扭蛋机独立组件

将扭蛋机主体抽成独立组件，页面只负责传入业务状态和处理事件。

组件暴露属性：

| 属性 | 说明 |
| --- | --- |
| `energy` | 当前能量石数量 |
| `cost` | 单次许愿消耗，默认 `20` |
| `disabled` | 外部禁用态，承接活动未开始、页面全局锁定等业务限制 |
| `capsules` | 扭蛋展示配置，本期仅前端本地配置图片数量、位置和类型 |
| `minDuration` | 基础动画时长，保证接口过快返回时仍播放完整核心动画 |
| `timeout` | 接口超时时间，按 PRD 固定 `5000ms` |

组件内部维护视觉态，不直接扣能量、不直接修改奖品资产。业务结果通过事件交给页面处理，避免组件和活动账户、任务体系耦合。

### 2. 建立按钮与许愿状态机

按钮态不使用多个布尔值拼装，统一由状态机派生：

| 状态 | 触发条件 | 按钮表现 |
| --- | --- | --- |
| `ready` | `energy >= cost` 且未许愿中 | 允许点击，文案“能量充足，20能量/次” |
| `insufficient` | `energy < cost` 且未许愿中 | 置灰，文案“能量不足，20能量/次” |
| `drawing` | 已点击许愿，接口与动画未完成 | 许愿中，禁止重复点击 |

点击逻辑：

1. `insufficient` 状态点击只提示“能量石数量不足，快去做任务获取更多吧”，不启动动画。
2. `ready` 状态点击后进入 `drawing`，立即触发组件 `start` 事件，并启动机内混合动画与接口请求。
3. `drawing` 状态下屏蔽重复点击，并由页面根据 `start` 事件阻止其他外部交互。

### 3. 接口请求与动画时序解耦

后端决定真实抽奖结果，前端只负责展示固定出口扭蛋和结果映射。实现上将一次许愿拆成两个并行任务：

1. 动画任务：播放扭蛋机启动、机内混合，至少持续 `minDuration`。
2. 接口任务：请求后端抽奖接口，最长等待 `5000ms`。

收口规则：

1. 接口成功且基础动画未结束：等待基础动画结束后再播放出口扭蛋滚出和弹窗。
2. 接口成功且基础动画已结束：直接进入出口扭蛋滚出和弹窗。
3. 接口失败、接口超时、网络断开：不播放出口扭蛋滚出，不展示奖品弹窗；机内动画缓慢停止并复位，toast 提示“许愿失败，请检查网络后重试”。

以“接口请求”和“基础动画”两个并行任务记录状态，并在二者满足条件后统一收口：

```text
开始许愿：
  进入许愿中
  启动机内混合动画
  同时发起抽奖请求和 5s 超时计时

当接口成功：
  记录奖品结果
  若基础动画已完成，则进入成功收口
  若基础动画未完成，则等待基础动画完成

当基础动画完成：
  若已有接口成功结果，则进入成功收口
  若接口仍未返回，则继续保持低速运行

成功收口：
  抛出成功事件
  播放出口扭蛋动画
  打开奖品弹窗
  抛出动画结束事件

失败与超时：
  清理本轮计时
  缓慢停止并复位
  抛出失败事件
```

失败时需要清理本次所有计时器，取消后续阶段回调，并通过本轮请求标识防止旧请求返回后污染当前状态。

### 4. 扣能量时机由页面控制

组件不在点击瞬间扣能量。组件在接口成功后抛出 `success` 事件，页面确认服务端返回成功后再更新能量展示。

该扣能量时机满足：

1. 超时、断网、接口失败时不扣代币。
2. 后端奖品结果和能量流水一致。
3. 页面在 `success` 后统一刷新用户资产、任务进度和埋点。

组件事件：

| 事件 | 时机 | 用途 |
| --- | --- | --- |
| `start` | 许愿动画开始 | 页面加锁、埋点、关闭其他浮层 |
| `success` | 接口成功返回 | 扣除能量、缓存奖品结果 |
| `fail` | 接口失败、接口超时、网络断开 | toast、恢复外部交互 |
| `animationend` | 出口扭蛋与结果弹窗准备完成 | 页面解锁并继续后续引导 |

### 5. CSS 扭蛋动画实现

产品扭蛋机大图由 UI 提供完整图片，前端不拆分组合机身。机内扭蛋数量、位置、层级、轻微旋转角度本期写在前端本地配置中，便于还原设计稿效果。

出口动画固定使用 UI 提供的通用扭蛋图，不根据接口奖品变化切换出口扭蛋图片。接口结果只影响奖品弹窗内容，避免“出口球”和真实奖品资源之间产生不必要的素材依赖。

扭蛋动画使用 CSS 动画完成，逻辑层只切换阶段 class 和控制阶段时间。机内每颗扭蛋使用独立轨迹动画，轨迹通过 `transform: translate/rotate/scale` 完成，避免频繁修改布局属性。不同扭蛋配置不同初始位置、层级、动画延迟和轨迹关键帧，形成错位混合效果。

动画阶段拆分为：

1. `idle`：机内扭蛋轻微浮动。
2. `starting`：按钮按下后旋钮转动，机内扭蛋从 idle 切换到高速混合。
3. `mixing`：扭蛋按预设轨迹循环运动，基础动画时长覆盖一次完整混合过程。
4. `waiting`：基础动画完成但接口未返回时，扭蛋切换为低速循环，保持许愿中状态。
5. `exporting`：接口成功且基础动画结束后，机内动画停止，固定扭蛋从出口滚出。
6. `opening`：出口动画结束后展示弹窗中的扭蛋打开动效。
7. `result`：展示奖品内容。
8. `resetting`：失败时从当前动画状态缓慢停止并复位。

动画时长在逻辑层集中维护，并和样式动画时长保持一致，避免视觉阶段和逻辑阶段错位。

动画时间线：

| 时间点 | 阶段 | 实现逻辑 |
| --- | --- | --- |
| `T0` | `starting` | 点击许愿后立即切换启动 class，旋钮转动，组件抛出 `start` 事件 |
| `T0 - T基础动画结束` | `mixing` | 机内扭蛋播放 CSS 轨迹动画，每颗扭蛋使用不同关键帧和延迟 |
| `T基础动画结束后接口未返回` | `waiting` | 机内扭蛋切换为低速 CSS 循环，表现为缓慢等待 |
| `接口成功且基础动画已完成` | `exporting` | 停止机内混合，播放出口扭蛋位移和滚动动画 |
| `出口动画结束` | `opening` | 打开奖品弹窗，先展示扭蛋打开动效 |
| `打开动效结束` | `result` | 切换为奖品内容，抛出 `animationend` 事件 |
| `接口失败、接口超时、网络断开` | `resetting` | 停止成功链路，切换复位 class，扭蛋和旋钮缓慢回到初始状态 |

伪代码流程：

```text
点击许愿：
  stage = starting
  添加启动 class
  下一帧切换 stage = mixing
  添加机内扭蛋混合 class

基础动画完成：
  若接口已成功：
    stage = exporting
    添加出口扭蛋滚出 class
  若接口未返回：
    stage = waiting
    添加低速等待 class

接口成功：
  缓存奖品结果
  若基础动画已完成：
    stage = exporting
    添加出口扭蛋滚出 class

出口动画完成：
  stage = opening
  打开奖品弹窗并播放扭蛋打开 class

打开动画完成：
  stage = result
  展示奖品内容
  抛出动画结束事件

失败与超时：
  清理后续动画计时
  stage = resetting
  添加缓停复位 class
  复位结束后根据能量回到 ready/insufficient 状态
```

CSS 动画职责边界：

1. 扭蛋位置、旋转、缩放、层级错位均由 CSS 关键帧完成。
2. 逻辑层不逐帧计算动画，只控制阶段 class、计时器和事件抛出。
3. 出口扭蛋动画独立于机内扭蛋动画，成功前不渲染出口扭蛋。
4. 失败链路只播放复位动画，不进入出口滚出和弹窗打开阶段。
5. 弹窗中的扭蛋打开动画独立于抽奖结果，奖品内容在打开动画结束后切换展示。

### 6. 奖品数据映射与弹窗展示

奖品基础数据仍存放在前端本地配置，后端接口只返回奖品标识、兑换码等动态字段。页面收到接口结果后按奖品标识查找本地配置，组装弹窗展示数据。

本地配置描述奖品名称、奖品图片、是否需要兑换、规则类型等静态信息；接口结果补充兑换码、中奖记录 ID 等动态信息。

弹窗展示：

1. 展示奖品大图、【收下】、【晒一下】按钮。
2. `needRedeem` 为 `true` 时展示兑换入口和规则入口。
3. 兑换、规则点击逻辑复用既有阅读活动能力，只在本模块透传奖品类型和兑换参数。
4. 抽中即默认收下，点击【收下】仅关闭弹窗，不再发额外领取请求。

接口返回的奖品标识无法命中本地配置时，按失败流程处理：复位动画并提示失败，避免展示空奖品。

### 7. 分享弹窗与微信分享页

点击【晒一下】时跳转微信分享页，并带上奖品标识、活动标识等必要参数。分享文案固定为：

```text
Hi～加入企鹅读伴，一起做任务赢好礼！
```

分享页负责配置分享内容，扭蛋组件仅抛出点击事件，奖品弹窗承接跳转逻辑。

### 8. 背景 Lottie 动效接入

背景动效使用设计师提供的 Lottie 帧动画资源，由页面层接入，不放入扭蛋机核心组件，避免背景表现影响组件复用。

小程序接入方式：

1. 引入项目统一的 Lottie 小程序渲染组件。
2. 在页面背景层放置 Lottie 组件，层级低于扭蛋机主体。
3. 页面完成初始化后加载动画资源，设置循环播放。
4. 页面隐藏和卸载时暂停并销毁动画实例，避免后台持续渲染。
5. 动画资源加载失败时保留静态背景图，不影响扭蛋主流程。

Lottie 仅承接背景氛围动效；扭蛋启动、出口滚动、弹窗打开仍使用本地 CSS 动画，以保证和接口状态机精确同步。

### 9. 异常与并发保护

关键保护点：

1. 点击后立即进入 `drawing`，防止连点产生多次请求。
2. 每次许愿生成唯一请求标识，计时器、接口回调、动画结束回调均校验当前请求标识。
3. 页面卸载、弹窗关闭、失败复位时统一清理计时器。
4. 超时后即使真实接口稍后返回，也不得再播放出口动画，不得覆盖当前奖品。
5. 失败复位期间继续保持组件锁定，复位完成后根据剩余能量回到对应非许愿态。
